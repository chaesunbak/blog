작년 4월에 AWS Certified Cloud Practitioner 취득했고 이어서 올해 4월에 AWS Certified Developer - Associate 취득했다. 시험을 준비하면서 얻은 팁을 정리하고 공유한다.

## 시험정보

시험 응시료 할인 바우처, 학습자료 등 AWS 자격증 관련 최신정보는 [레딧](https://www.reddit.com/r/AWSCertifications/)이 가장 좋다. 가장 활발한 커뮤니티인 것 같다.

## 학습방법

### 공식 가이드

[공식 가이드](https://aws.amazon.com/ko/certification/certified-developer-associate/)를 먼저 읽어보기를 추천한다.

### AWS Skill Builder

[스킬 빌더](https://skillbuilder.aws/category/exam-prep/developer-associate-DVA-C02)에서 샘플 문제 조금과 강의 조금을 무료로 제공해준다. 구독 옵션도 존재하지만 무료 강의로 충분한 것 같다.

### 덤프

기존에 공부했던 덤프 사이트에 광고가 많이 뜨고 불편해서 이번에는 유튜브로 덤프를 풀었다. 시험명을 검색하면 바로 찾을 수 있다.

가장 쉽고 빠르게 시험에 합격하는 방법은 덤프를 많이 푸는 것 같다. 문제를 풀면서 해설을 꼼꼼히 읽어보고 모르거나 헷갈리는 부분은 공식문서를 찾아보거나 AI에게 물어보면서 개념들을 정리하면 좋을 것 같다.

## 시험 팁

신분증과 영어이름이 포함된 체크카드 또는 신용카드가 필요하다.

오프라인으로 응시했다. 시험장이나 상황마다 다를 것 같은데 나는 예약한 시간보다 30분 먼저 도착했는데 일찍 시험을 시작할 수 있었다.

객관식 시험인 만큼 모든 선지를 꼼꼼히 읽어보면서 소거법으로 풀면 좋을 것 같다.

## 후기

사실 AWS 자격증은 실제로는 없는거고, 아마존이 교모하게 사람들을 가스라이팅해서 본인들 서비스 설명 + 홍보를 자격증이라고 속이면서 돈을 벌고 있는거라고 생각하게 된다. 응시료 말고 더 가치있게 쓸 수 있는 방법이 많은 것 같다.

그래도 굳이 장점을 찾아 이야기해보자면, **AWS의 다양한 서비스들을 체계적으로 여러 관점(특히 보안영역을 포함해서)에서 시험이라는 익숙한 형태로 공부할 수 있는 것이 장점인 것 같다.** 평범한 사이드프로젝트에서는 EC2, RDS, S3같은 AWS의 기본적인 서비스만 사용하게 되는것 같은데, 엔터프라이즈급 요구사항을 SQS, SNS, Lambda Function, API Gateway, Elastic Cache 같은 서비스를 사용해서 간단하면서도 확장가능하게 구축하는 방법을 익힐 수 있어서 만족스럽다.

대 AI 시대에 단순 지식을 평가하는 시험이 무슨 의미가 있는가 싶기도 하지만, 이런게 있다고 익혀두면 AI로 그걸 구체화하고 실현하는 것이 쉬워져서 의미가 완전 없지는 않은 것 같다.
